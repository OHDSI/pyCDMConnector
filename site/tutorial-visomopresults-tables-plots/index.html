    visOmopResults: Tables and plots¶

    This tutorial summarises the R visOmopResults vignettes on creating publication-ready tables and plots from data in summarised_result format (and from arbitrary data.frames). The package is used across the OHDSI/DARWIN ecosystem to present cohort characteristics, incidence/prevalence, and other results.

    Introduction¶

    visOmopResults provides:

    • Tables: Formatted tables from summarised_result or any data.frame (gt, flextable, tinytable, datatable, reactable).
    • Plots: Scatter, box, bar, and other plot types from summarised_result (and compatible data), with consistent styling and faceting.

    Although the package targets summarised_result (see Summarised result), most functions work with any data.frame that has the expected columns (e.g. estimate_name, estimate_value, group/strata columns).

    Tables¶

    Main function: visTable()¶

    • result – A data.frame or summarised_result.
    • groupColumn – Column(s) to group rows by (e.g. sex, cohort_name).
    • rename – Map column names to display labels.
    • type – Output type: "gt", "flextable", "tinytable", "datatable", "reactable".
    • hide – Columns to hide from the table.
    • estimateName – Combine or rename estimate columns (e.g. "<bill_length_mm> - <bill_depth_mm>" for a combined label).
    • header – Column(s) to use as header levels (e.g. species, island).

    If the data has the standard estimate columns (estimate_name, estimate_type, estimate_value), visTable can format numbers (e.g. 2 decimals), combine estimates with estimateName, and build multi-level headers with header.

    visOmopTable()¶

    Higher-level function tailored to summarised_result: uses the object’s group/strata/additional structure and settings to produce a formatted table with minimal options. Wraps the same formatting building blocks as visTable.

    Formatting building blocks¶

    The package also exposes lower-level format* functions (e.g. formatHeader, formatEstimateValue, formatEstimateName) so you can compose custom table pipelines (rename, round, group, then render with gt/flextable/etc.).

    Plots¶

    Principles¶

    1. Tidy format – Plotting functions expect tidy summarised_result: group/strata/additional name-level pairs split into separate columns (see Summarised result#tidy). Use tidyColumns() to see which columns are available for plotting.
    2. Subset by variable – Filter to one (or a few) variable_name before plotting; avoid mixing unrelated variables in one plot.

    Main plot functions¶

    • scatterPlot() – x, y, optional ymin/ymax (e.g. for ribbons), line, point, ribbon, facet, colour, group.
    • boxPlot() – Boxplots by group/strata; facet, colour, x.
    • barPlot() – Bar charts; facet, colour, x.
    • densityPlot() – Density curves (e.g. for density estimates in summarised_result).

    Styling¶

    • style – e.g. "default" or "darwin" (branded style).
    • themeVisOmop() – Apply theme to a ggplot; can set font size and other options.
    • For “darwin” style with Calibri, the R package can use extrafont; in Python you would use matplotlib/plotly themes or custom CSS for web.

    Quarto reports and Shiny apps¶

    visOmopResults vignettes also cover:

    • Quarto reports – Embedding visTable and visOmop plot output in Quarto (and R Markdown). In Python, the same tables/plots can be generated with pandas/matplotlib/plotly and embedded in Jupyter, Quarto, or Streamlit.
    • Shiny apps – Using visTable with reactable or datatable for interactive apps. In Python, Streamlit or Dash can host similar interactive tables and plots.
    • Style – Centralised styling (fonts, colours) for tables and plots; in Python you’d use a shared theme or config.

    In Python¶

    There is no visOmopResults port yet. You can:

    1. Tables: Use pandas DataFrames in the same shape as tidy summarised_result; format with pandas (round, rename), then render with tabulate, DataFrame.to_html(), or export to CSV/Excel. For interactive tables, use itables or streamlit dataframe display.
    2. Plots: Use matplotlib or plotly with the same variables (x, y, facet, colour). Filter to one variable_name, use tidy columns for x/facet/colour, and map estimate_name to y (e.g. median, q25, q75 for a box/ribbon).
    3. Keep column names and structure aligned with summarised_result and tidy format so that any future Python visOmopResults-style package or R export/import can reuse the same data.
