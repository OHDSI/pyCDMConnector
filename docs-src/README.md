# CDMConnector documentation (Quarto, pkgdown-style)

This directory contains the **source** for the CDMConnector documentation website. Rendered HTML is written to **`/docs`** and served by GitHub Pages.

## Setup (Quarto)

- **Source**: All Quarto inputs live in **`docs-src/`** (`.qmd`, notebooks, assets).
- **Output**: Running `quarto render docs-src/` writes HTML (and assets) to **`../docs`** (i.e. repo root `docs/`).
- **API reference**: Generated by **quartodoc** from the `cdmconnector` package. Quartodoc is configured in `_quarto.yml` under the `quartodoc` key (no separate `quartodoc.yml`).

## Local build workflow

```bash
# From repo root

# 1. Optional: create venv and install package + docs deps
python -m venv .venv && source .venv/bin/activate  # or use pyenv
pip install -e ".[docs]"

# 2. Generate API reference (writes into docs-src/reference/)
quartodoc build

# 3. Render website into /docs
quarto render docs-src/
```

After a successful build:

- `docs/index.html` — homepage
- `docs/notebooks/` — rendered tutorial notebooks
- `docs/reference/` — API reference (from quartodoc)
- `docs/contributing/`, `docs/development/` — contributing and development pages

**HTML in `docs/` is committed** so GitHub Pages (source: branch `main`, folder `/docs`) can serve the site without a build step in CI, unless you choose to build in CI and deploy the artifact.

## Structure

| Path | Purpose |
|------|---------|
| `docs-src/_quarto.yml` | Quarto website config, nav, quartodoc config |
| `docs-src/index.qmd` | Homepage |
| `docs-src/notebooks/` | Tutorial Jupyter notebooks + `index.qmd` |
| `docs-src/reference/` | quartodoc-generated API pages + `index.qmd` |
| `docs-src/contributing.qmd` | Contributing guide |
| `docs-src/development.qmd` | Developer workflow, release checklist |
| `docs-src/assets/` | Images, cohort JSON, etc. |

## Design

- **pkgdown-style**: Source in one dir, rendered output in another; output is committed.
- **Narrative-first**: Tutorials (notebooks) and prose before API.
- **Reproducible**: Notebooks can be run locally; Quarto renders without executing by default unless configured otherwise.
